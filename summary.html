2. The Math

2.1 Full-Vocabulary Proxy Tuning

Given logits from three models over vocabulary $V$ (size 151,936):

$$\text{score}_{\text{full}}(i) = s_{\text{large}}(i) \;+\; \lambda \cdot \bigl(s_{\text{ft}}(i) - s_{\text{raw}}(i)\bigr) \qquad \forall\, i \in V$$

The recovered probability distribution is $p_{\text{rec}} = \text{softmax}(\text{score}_{\text{full}})$.

Oracle identity: when $s_{\text{large}} = s_{\text{raw}}$ and $\lambda = 1$:

$$\text{score}_{\text{full}}(i) = s_{\text{large}}(i) + s_{\text{ft}}(i) - s_{\text{large}}(i) = s_{\text{ft}}(i)$$

So $p_{\text{rec}} = p_{\text{ft}}$ exactly. $\text{KL}(p_{\text{ft}} \| p_{\text{rec}}) = 0$ — by algebra, not by approximation.

2.2 Restricted Renormalized Delta

Let $S_t = \text{top-}k(s_{\text{large}})$ be the candidate set. For $i \in S_t$:

$$\delta(i) = \underbrace{\bigl[s_{\text{ft}}(i) - \text{LSE}(s_{\text{ft}}[S_t])\bigr]}_{\log p_{\text{ft}}^{S_t}(i)} - \underbrace{\bigl[s_{\text{raw}}(i) - \text{LSE}(s_{\text{raw}}[S_t])\bigr]}_{\log p_{\text{raw}}^{S_t}(i)}$$

where $\text{LSE}(x) = \max(x) + \log\sum_j \exp(x_j - \max(x))$ (numerically stable log-sum-exp).

Apply the delta to the large model's full-vocab log-probabilities:

$$\text{score}_{\text{restricted}}(i) = \log p_{\text{large}}^{V}(i) + \lambda \cdot \delta(i) \qquad i \in S_t$$

Then renormalize: $p_{\text{rec}}^{S_t} = \text{softmax}(\text{score}_{\text{restricted}})$ over $S_t$ only.

2.3 Why the Oracle Guarantee Survives Restriction

When $\delta = 0$, the score is just $\log p_{\text{large}}^V(i)$ for $i \in S_t$. Renormalizing over $S_t$:

$$p_{\text{rec}}^{S_t}(i) = \frac{p_{\text{large}}^V(i)}{\sum_{j \in S_t} p_{\text{large}}^V(j)} = \frac{e^{s_{\text{large}}(i)} / Z}{\sum_{j \in S_t} e^{s_{\text{large}}(j)} / Z} = \frac{e^{s_{\text{large}}(i)}}{\sum_{j \in S_t} e^{s_{\text{large}}(j)}} = p_{\text{ft}}^{S_t}(i)$$

The normalizing constant $Z$ cancels. Mathematically, restricted KL = 0 in oracle mode.

3. Worked Examples

Example A — Oracle with toy vocabulary (4 tokens)

$V = \{A, B, C, D\}$, all models identical:

$$s = [3.0,\; 1.0,\; 0.5,\; -1.0]$$

Full vocab: $\text{score} = s + 1\cdot(s - s) = s$, so $p_{\text{rec}} = p_{\text{ft}}$. KL = 0. ✓

Restricted to $S_t = \{A, B\}$ (top-2):

LSE([3.0, 1.0]) = 3.0 + log(1 + exp(-2)) = 3.1269

log p_ft(A) = 3.0 - 3.1269 = -0.1269    →  p_ft(A) = 0.8808
log p_ft(B) = 1.0 - 3.1269 = -2.1269    →  p_ft(B) = 0.1192

delta = [0, 0]  (ft = raw)

log p_large_V:
  LSE([3.0, 1.0, 0.5, -1.0]) = 3.3390
  log p_large(A) = -0.3390
  log p_large(B) = -2.3390

score_restricted = [-0.3390, -2.3390] + [0, 0]

Renormalize:
  LSE([-0.3390, -2.3390]) = -0.2121
  log p_rec(A) = -0.3390 + 0.2121 = -0.1269  ← matches log p_ft(A) !
  log p_rec(B) = -2.3390 + 0.2121 = -2.1269  ← matches log p_ft(B) !

KL = 0 ✓ — The $Z = \exp(3.3390)$ cancels in the ratio.

Example B — Non-oracle: fine-tuning shifts mass from A to B
$$s_{\text{large}} = s_{\text{raw}} = [3.0,\; 1.0,\; 0.5,\; -1.0]$$ $$s_{\text{ft}} = [1.0,\; 4.0,\; 0.5,\; -1.0]$$

Fine-tuning suppressed $A$ ($3 \to 1$) and boosted $B$ ($1 \to 4$).

Full vocab:

score = [3, 1, 0.5, -1] + ([1, 4, 0.5, -1] - [3, 1, 0.5, -1])
      = [3-2, 1+3, 0.5, -1]
      = [1.0, 4.0, 0.5, -1.0]  ← recovers s_ft exactly

KL = 0 ✓ (works whenever $M_{\text{large}} = M_{\text{raw}}$, regardless of $M_{\text{ft}}$).

Restricted to $S_t = \{A, B\}$:

delta(A) = [1.0 - LSE(1,4)] - [3.0 - LSE(3,1)]
         = [1.0 - 4.0486] - [3.0 - 3.1269]
         = -3.0486 - (-0.1269)
         = -2.9217   ← large negative: ft suppressed A

delta(B) = [4.0 - 4.0486] - [1.0 - 3.1269]
         = -0.0486 - (-2.1269)
         = +2.0783   ← large positive: ft boosted B

score(A) = -0.3390 + (-2.9217) = -3.2607
score(B) = -2.3390 + (+2.0783) = -0.2607

Renormalize over {A, B}:
  p_rec(A) = exp(-3.0486) = 0.0474
  p_rec(B) = exp(-0.0486) = 0.9526

p_ft^{S_t}(A) = exp(1)/[exp(1)+exp(4)] = 0.0474
p_ft^{S_t}(B) = exp(4)/[exp(1)+exp(4)] = 0.9526

Exact match. KL = 0 ✓

Example C — Cross-model transfer (restriction costs something)

Now $M_{\text{large}} \neq M_{\text{raw}}$ — the setting we actually care about:

$$s_{\text{large}} = [5.0,\; 2.0,\; 1.0,\; 0.0]$$ $$s_{\text{raw}} = [3.0,\; 1.0,\; 0.5,\; -1.0]$$ $$s_{\text{ft}} = [1.0,\; 4.0,\; 0.5,\; -1.0]$$

Full vocab:

score = [5, 2, 1, 0] + ([1, 4, 0.5, -1] - [3, 1, 0.5, -1])
      = [3.0, 5.0, 1.0, 0.0]

p_ft   = softmax([1, 4, 0.5, -1])  = [0.0430, 0.8638, 0.0261, 0.0058]
p_rec  = softmax([3, 5, 1, 0])     = [0.1093, 0.8076, 0.0148, 0.0054]

KL(p_ft || p_rec) = 0.0430·log(0.0430/0.1093) + 0.8638·log(0.8638/0.8076)
                   + 0.0261·log(0.0261/0.0148) + 0.0058·log(0.0058/0.0054)
                 ≈ 0.0462

Non-zero KL! The cross-model transfer isn't perfect because the large model has a different base distribution than the raw model.

With restriction to $k=2$: we lose tokens $C$ and $D$ entirely. If fine-tuning shifted mass toward them, that signal is lost. The restricted KL will be different (potentially larger or smaller) than the full-vocab KL.

This is the fundamental tradeoff GRAFT explores: smaller $k$ means faster inference but potentially worse transfer quality.

4.3 Interpreting the Non-Zero Restricted KL

precision artifact

Mathematically, all restricted KL values should be exactly 0 in oracle mode (the $Z$ from full-vocab log_softmax cancels; see Section 2.3). The non-zero values — including negative ones — reveal float32 precision limits:

  • The log_softmax(s_large) step sums $\exp(\cdot)$ over 151,936 tokens, accumulating rounding error $\varepsilon$
  • The direct log_softmax(s_large[S_t]) sums only $k$ tokens, with negligible error
  • Mathematically the paths are equivalent ($Z$ cancels), but float arithmetic is not associative
  • Negative KL values are impossible for true distributions — they confirm this is pure precision noise

Practical implication: in the real cross-model experiment, any restricted KL below ~0.15 nats should be treated as indistinguishable from zero.

6.3 Key Insight: Restriction is Free When large = raw

The restricted KL values remain near-zero (precision noise only) at all $k$ values, despite the large delta. This is because the $M_{\text{large}} = M_{\text{raw}}$ identity makes the formula algebraically exact:

$$\text{score}(i) = \log p_{\text{large}}^V(i) + \lambda \cdot \delta(i) = \log p_{\text{raw}}^V(i) + \delta(i)$$

After renormalization over $S_t$, the $Z$ from full-vocab softmax cancels regardless of the delta magnitude. The real test of restriction cost requires Step 3, where $M_{\text{large}} \neq M_{\text{raw}}$.

7.3 Key Insight: Restriction as Regularization

Smaller $k$ produces BETTER results — the opposite of what naive intuition suggests.

KL(ft || recovered) at k=10 is 0.017 vs 0.758 at k=full. Restricting to the large model's top candidates acts as beneficial regularization: it filters out noisy delta values for tokens that the large model assigns near-zero probability, leaving only the high-confidence tokens where the delta carries clean signal.

This is practically very useful: k=10–50 is both the cheapest to compute AND the most accurate.

12. Decisive Conclusion: What GRAFT Means for You

The Core Finding

You can make a large base model follow instructions without fine-tuning it, by borrowing the behavioral delta from a smaller model pair that shares the same tokenizer.

$$\underbrace{p_{\text{steered}}}_{\text{14B that follows instructions}} = \text{softmax}\Bigl( \underbrace{s_{\text{14B-base}}}_{\text{large model logits}} + \lambda \cdot \underbrace{\bigl(\log p_{\text{8B-inst}}^{S_t} - \log p_{\text{8B-base}}^{S_t}\bigr)}_{\text{delta: what instruction-tuning learned}} \Bigr)$$

What This Enables

  1. Instant instruction-tuning for new large models. When a new base model is released in the same family (e.g., Qwen3-32B-Base), you don't need to wait for the instruct variant. Graft the 8B instruct delta onto it immediately. The delta transferred at 85.8% top-1 accuracy across 541 IFEval prompts, and achieved 59.7% IFEval strict accuracy — recovering 96.1% of the 14B instruct's 62.1%, up from 41.0% base.

  2. No GPU training required. The entire method runs at inference time. No fine-tuning, no gradient computation, no training data. Just three forward passes per token. On an M3 Pro, delta-steered generation runs at ~6.7 tok/s (vs 13.7 tok/s for the base model alone — a 2x overhead).

  3. Restriction makes it better AND cheaper. Restricting the delta to just the top-$k$ tokens from the large model acts as regularization:

    $$S_t = \text{top-}k(s_{\text{large}}) \qquad \delta(i) = \log p_{\text{ft}}^{S_t}(i) - \log p_{\text{base}}^{S_t}(i) \quad \forall i \in S_t$$

    At $k = 10$: 92.1% top-1 match (better than $k = 200$ at 87.1%). The large model's top-$k$ concentrates the delta on tokens that matter, filtering out noise on improbable tokens. And computing the delta over 10 tokens instead of 151,936 is 15,000x cheaper.
