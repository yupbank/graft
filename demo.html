The Restriction Problem

Top-50 restriction only considers tokens ranked #1–#50 by the base model (before applying the delta):

Full-vocab proxy-tuning → sees ALL 151,936 tokens

Picks de (total=24.19) → generates "dear friend, ..."

Concise letter, 47 words → PASSES length constraint

Top-50 restriction → only sees base model's top-50

de is at rank #79 — invisible. Can't pick it.

Picks Hi (total=9.03, best available) → generates "hello my friend, ..."

Rambling letter, 183 words → FAILS length constraint

The Fundamental Mechanism

Instruction-tuning teaches models to use tokens that base models ignore.

  • "Write a letter" → dear (rank #79 in base)
  • "Respond in JSON" → { (could be rank #200+ in base)
  • "Answer in French" → French tokens (very low in English base)

The delta's job is to promote these tokens. But top-k restriction blocks the promotion when the target is below rank k.

The 3pp gap (59% → 62%) = the fraction of generation steps where the instruct delta needs to promote a token from outside the base model's top-50. It happens ~3% of the time, but each occurrence can cascade into a different-length response that fails a constraint.
